Conduit
Automotive Service Management Platform
See it live Repository
Why
Indian automotive aftermarket handles ~50M service jobs annually. Most dealerships still rely on paper records and manual parts lookup — a slow, error-prone process that limits advisor throughput and damages customer experience.
What it does

5-agent LangGraph pipeline — Intake, Inventory, Quoting, Transaction, Replenishment — autonomously turns a customer complaint into a GST-compliant repair order

Service intake time reduced from 25 min to 45 sec (~97% reduction)

Potential to improve advisor throughput by 3–5x at the same headcount

HITL interrupts pause the pipeline for supervisor approval on EV jobs, high-value quotes, and low-confidence classifications

LangSmith observability traces every agent step with token cost, latency, and decision data — full pipeline visibility in production

Tradeoffs and Risks

LLM over rules: fault classification requires understanding varied natural language complaints — rule-based systems fail on edge cases and non-standard descriptions

Latency managed via SSE streaming so advisor sees real-time pipeline progress; eval cost controlled via two-layer framework (~$1.05 per full run)

ShopMind
Real-Time Manufacturing Intelligence
See it live Repository
Why
Unplanned downtime costs Indian manufacturers ~INR 12,000 per production line per hour. Most plants rely on shift-end reports — by the time a problem surfaces, the damage is already done.
What it does

Four specialised agents watch production lines continuously: OEE tracking (every 2 min), maintenance prediction with LLM narratives (every 10 min), compound crisis detection (every 5 min)

Text-to-SQL Copilot lets plant managers ask plain-language questions — "Why is Line 3 slow?" — and get grounded, data-backed answers instantly

Compound crisis detection catches when 3+ machines on the same line fail within 20 min — a pattern invisible to individual alert monitors

HITL action log lets supervisors acknowledge, escalate, or suppress alerts with full audit trail per shift

LangSmith observability captures every LLM call with token cost, latency, and full prompt trace — enabling ongoing cost and quality monitoring in production

Architecture
LLM
GPT-4o — maintenance narratives, compound crisis reasoning, Text-to-SQL Copilot
↓
Agents
Blackboard pattern — 4 agents coordinate via shared PostgreSQL, no direct coupling
↓
Eval
LLM-as-Judge BIRD-SQL rubric — 12 Copilot + 3 agent health checks, ~$0.10/run
↓
Infra
GCP Cloud Run + Neon PostgreSQL — LangSmith traces, GitHub Actions CI/CD
Tradeoffs and Risks

OEE calculation is pure Python (not LLM) — high-frequency operations every 2 min make LLM usage prohibitively expensive; LLM reserved for narratives and compound reasoning only

SQL hallucination in Copilot mitigated by read-only DB enforcement and LLM-as-Judge BIRD-SQL rubric (GPT-4o judge, avg 4.5/5 score); LangSmith tracks token cost and latency per Copilot query in production

Alert fatigue controlled by dedup logic — max one alert per machine per type per shift — preventing cascading noise from repeated triggers
